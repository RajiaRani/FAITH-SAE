# TWO PROFILES — read this before installing
# -----------------------------------------------------------------------------
#   (A) SMOKE  (this build machine, CPU, NO GPU, NO open_clip):
#       Every entry script here has a `--smoke` flag that runs a tiny CPU path on
#       synthetic-but-real-SHAPED tensors. It needs ONLY the "core" block below,
#       all of which already ships with /usr/bin/python3 on this machine — so you
#       can self-test the whole pipeline today with ZERO installs:
#           /usr/bin/python3 run_all_real.sh ... (or each *.py --smoke)
#       In particular open_clip is GUARDED by try/except in data_real.py, so every
#       module imports cleanly on CPU without it.
#
#   (B) REAL  (a single A100/H100 80GB box):
#       Install EVERYTHING below into a fresh venv/conda env. The "real-only"
#       block (open_clip_torch, datasets, webdataset, wandb) pulls large model
#       weights and is the part the smoke path never touches.
#           python -m venv .venv && . .venv/bin/activate
#           pip install -r requirements_real.txt
#       See RUN_AT_SCALE.md for the matching hardware / dataset / time budget.
# =============================================================================
